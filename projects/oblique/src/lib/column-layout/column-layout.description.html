The column layout component comes with 1, 2 or 3 columns. Both side columns are collapsible / expansible and optional. They both use 20 % of the total width while the center column takes up the remaining 60%. The side columns can be enlarged with the wider option to use 40% each. This option is only suitable when only 2 columns are displayed.

It comes with preconfigured padding which can be canceled with the noLayout input. On smaller viewports, both side columns are moved below the main one. This behavior can be tweaked by altering their order property.

If the ObColumnLayout is set to a specific height, then it's overflow property has to be set to auto for it to work properly.

To adapt the column width to your needs, overwrite following CSS rules:

@media (min-width: 992px) {
	.column-layout {
		.column-left,
		.column-right {
			flex-basis: $column-width;
			max-width: $column-width;
		}
		.column-left.collapsed {
			margin-left: -$column-width;
		}
		.column-right.collapsed {
			margin-right: -$column-width;
		}
	}
}
