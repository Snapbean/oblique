The angular material Datepicker has been slightly restyled to match Oblique's look & feel.

Working with different locales is not straightforward. First, you have to register the locales you wish to use so that the datepicker know how to format a date. Then the MatMomentDateModule have to be preferred over the MatNativeDateModule because the native solution can't deal with alternate formats. Finally, the current locale has to be given to the date adapter so that the datepicker can format its date. This all can be achieved through the following code: 

// app.module.ts

import {registerLocaleData} from '@angular/common';
import localeFR from '@angular/common/locales/fr-CH';
registerLocaleData(localeFR);
// every module using the DatePicker

import {MatDatepickerModule} from '@angular/material/datepicker';
import {DateAdapter} from '@angular/material/core';
import {ObLanguageService} from '@oblique/oblique';

@NgModule({
	imports: [MatDatepickerModule, MatMomentDateModule, ...],
	...
}
export class MyModule {
	constructor(language: ObLanguageService, adapter: DateAdapter<any>) {
		language.setLocaleOnAdapter(adapter);
	}
}

Wrapper for the ng-bootstrap Datepicker component with some controls (clear, open & close). The validation is left to the containing form.

Do NOT apply form-control class on or-date-picker as it will break the layout.
