This service is used to get, set and add page meta information such as the content attribute of the <meta name="description"> and <title> tags. To activate it, simply import it in the AppModule.

Description

The description can be defined either globally on the ObDocumentMetaService or per route within the data object on routes definition: {path: 'home', component: HomeComponent, data: {description: 'Description'}}. The given string will be translated in both cases.

Each time a page is loaded, the description will be updated with the page specific one if provided or the global one otherwise.

Title

The title is composed of three parts: the page, a separator and a suffix. If either the page or the suffix is missing, then the separator will be omitted as well. If both the page and the suffix are missing, then the title won't be changed.

The page is defined per route within the data object on routes definition: {path: 'home', component: HomeComponent, data: {title: 'Title'}} and will be translated. If a route has no page defined, then the suffix will be show instead.

Both the separator and the suffix can be set on the ObDocumentMetaService and will be used on all pages. The suffix will also be translated and typically reflects the application's name.
