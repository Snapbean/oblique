A popover provides helpful contextual information in a dedicated container. It is meant for longer descriptions, help or additional actions that are not critical to the application.

The template can be rendered either right after its toggle or at the end of the body. In any case, the template is removed from the DOM when not active. Adding it after the toggle element preserves the tabindex order. Focusable components should receive focus in an order that preserves meaning and usability. See WCAG 2.0 Supporting Documents 2.4.3: Focus order. Unfortunately, this can lead to z-index problems that can only be solved by adding it at the end of the body.

Accessibility is ensured through the aria-describedby attribute and the tooltip role. The positioning is achieved with the popper library, for more information see popper.js.org.
